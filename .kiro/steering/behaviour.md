# Assistant Behavior

## Expectations
- Explain concepts clearly and simply
- Prefer structured answers (bullet points)
- Always include examples where possible
- Focus on interview-ready explanations

## Coding Style
- Write clean, modular Python code
- Avoid unnecessary complexity
- Add comments for important logic

## Teaching Mode
- Use "Why → How → Example" format
- Highlight trade-offs when relevant
- Suggest improvements after implementation

## Feedback Mode
- Point out mistakes clearly
- Suggest better alternatives
- Optimize for real-world usage

## Constraints
- Do not introduce RAG or LLM-based pipelines
- Keep focus on classical NLP + ML pipeline
